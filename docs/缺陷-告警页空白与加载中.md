# 缺陷：告警页面异常（加载中 / 白屏）

> 发现时间：2026-07-16
> 版本范围：v0.0.1（`sha-ade47cd`）
> 影响页面：`/#/alerts`、`/#/alerts/history`
> 优先级：🔴 P0 生产风险

---

## 现象

### Alerts 页（`/#/alerts`）

页面显示 `加载中…` 且永不消失。

### AlertHistory 页（`/#/alerts/history`）

页面完全白屏，`<div id="root">` 为空，无任何 React 内容渲染。

---

## 根因

两个问题同根：**Go `nil slice` JSON 序列化为 `null`，而前端 TypeScript 类型声明为 `T[]`（不含 `null`），未做防御性处理。**

### Bug 1 — `/#/alerts`

**后端** `web/api/alerts.go:18-25` `listRules()`：

```go
func (h *alertHandler) listRules(w http.ResponseWriter, r *http.Request) {
    rules, err := h.store.ListRules()  // 表空时返回 nil, nil
    ...
    writeJSON(w, rules)  // nil → JSON "null"
}
```

`web/alerts/store.go:164-192` `ListRules()`：`var out []AlertRule` 零值为 `nil`，无数据时不进 `rows.Next()` 循环，返回 `nil`。

**前端** `web/ui/src/pages/Alerts.tsx:92`：

```tsx
if (rules.loading || !rules.data) return <div className="loading">加载中…</div>
```

`rules.data` 是 `null` → `!null === true` → 永远显示加载中。

### Bug 2 — `/#/alerts/history`

**后端** `web/api/alerts.go:110-127` `listHistory()`：

```go
items, total, err := h.store.ListHistory(...)
// 表空时 items 为 nil, total 为 0
writeJSON(w, map[string]any{"total": total, "items": items})  // {"total":0,"items":null}
```

**前端** `web/ui/src/pages/AlertHistory.tsx:53`：

```tsx
{history.data.items.map(h => (...))}  // null.map(...) → TypeError → React 崩溃
```

## 所属缺陷模式

已在 [docs/前端架构负债.md](./前端架构负债.md) 🔴 生产风险「API 类型声明与运行时不一致」中记录：

> Go `[]T` 零值为 `nil`，JSON 序列化出 `null`，TypeScript 却声明为 `T[]`（不包括 `null`）。前端对 `null` 无防御 → 页面白屏/崩溃。

---

## 修复方案

### 短期（选择一项或同时做）

**方案 A：后端初始化为空切片（已在 `handleHosts`/`probeSeries` 使用）**

修改 `ListRules()` 和 `ListHistory()` 返回 `[]T{}` 而非 `nil`：

```go
// ListRules — 返回初始化的空切片
var out []AlertRule
→ 改为：
out := []AlertRule{}

// ListHistory — 同
var out []HistoryItem
→ 改为：
out := []HistoryItem{}
```

涉及文件：
- `web/alerts/store.go:174` — `var out []AlertRule` → `out := []AlertRule{}`
- `web/alerts/store.go:427` — `var out []HistoryItem` → `out := []HistoryItem{}`

**方案 B：前端对 `null`/`undefined` 做防御**

在 `Alerts.tsx` 和 `AlertHistory.tsx` 加空值保护：

```tsx
// Alerts.tsx — 改用可选链
{rules.data?.map(r => (...))}

// AlertHistory.tsx — 改用空数组默认值
{history.data?.items?.map(h => (...)) ?? null}
```

### 中期

引入 `zod` 运行时校验，API 响应解码时做类型断言 + 默认值（见 前端架构负债.md 方案）。

### 长期

从 Go 结构体自动生成 TypeScript 类型（`go-ts` / `typed-go`）。

## 受影响的 API 端点

| 端点 | 字段 | nil→null 风险 |
|---|---|---|
| `GET /alerts/rules` | 整个响应 (切片) | 🔴 本次触发 |
| `GET /alerts/history` | `items` 字段 | 🔴 本次触发 |
| `GET /hosts` | 整个响应 | 已用 `make([]hostOut, 0)` 修复 |
| `GET /events` | `points` 字段 | 已用 `[]vm.Point{}` 修复 |
| `GET /probes` | `.ping.up`, `.ping.duration` 等 | 已用 `[]vm.Point{}` 修复 |
| `GET /alerts/status` | `items` 字段 | ⚠️ 同模式，未修复 |
