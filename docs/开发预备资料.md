## 二、接口契约决策日志

### Event

原版 `Event` 三方法：`AsMapStr() common.MapStr`、`IgnoreCMDBLevel() bool`、`GetType() string`。

monitorbeat 决策：
- 砍 `AsMapStr`（libbeat 依赖）
- 砍 `IgnoreCMDBLevel`（BK 多层级下沉概念）
- 保留 `GetType`
- 新增 `GetData any`（替代 MapStr 通用承载）
- 新增 `GetTimestamp time.Time`（原版 timestamp 隐式由 libbeat 注入，需显式）
- 提供 `SimpleEvent` 实现 + `NewEvent(typ, data)` 便捷构造

### Task

原版 `Task` 方法 11 个（含 `GetBizID`）。

monitorbeat 决策：砍 `GetBizID`（业务概念）；保留 `GetTaskID / GetIdent`（调度器 reload dedup key）；`Reload/Wait/Stop/Run` 保留。

### TaskConfig

原版 `TaskConfig` 10 方法，含 `GetBizID/GetAvailableDuration/GetDataID/InitIdent`。

monitorbeat 决策：
- 砍 `GetBizID/GetAvailableDuration/GetDataID/InitIdent`
- 保留 `GetTaskID/GetIdent/SetIdent/GetTimeout/GetPeriod/GetType/GetLabels`
- 新增 `GetEnabled bool`（原版无 enabled 概念，依赖配置文件 include/exclude）
- `Clean() error` 保留作为 cleanParams 入口

### Config

原版 `Config` 3 方法，包含 `GetGatherUpDataID`。

monitorbeat 决策：
- 砍 `GetGatherUpDataID`
- 新增 `GetCheckInterval time.Duration`（原版 `configs.Config.CheckInterval` 字段，未走接口）

### Scheduler

原版 `Scheduler.Reload` 签名 `(ctx, Config, []Task) error` — 保留。

`define.BaseScheduler` 在原版是独立 struct，monitorbeat 合入 `define/scheduler.go` 文件，包含 6 个默认实现。

## 三、scheduler daemon 关键改造

### 1. utils.Atomic → atomic.Bool

原版 `utils.Atomic` 是布尔锁，monitorbeat 改 `atomic.Bool` + `CompareAndSwap`，零额外包依赖。

### 2. utils.RecoverFor → 显式 defer recover

原版用 `defer utils.RecoverFor(func(err) {...})` 包装，monitorbeat 改 `defer func() { if r := recover(); r != nil {...} }()`，零 libgse 依赖。

### 3. libgse/storage 删除

原版 `StroedIntervalJob.initStoredCheckTime` 用 `storage.Get` 持久化上次执行时间，monitorbeat P0 全删，重启后从 `time.Now()` 重新开始。P1 需要时补 file storage。

### 4. configs.Config 强转去除

原版 `Daemon.Start` 中 `globalConfig := s.Config.(*configs.Config)` 强转接口拿到 `CheckInterval`，monitorbeat 把 `GetCheckInterval` 加入 `define.Config` 接口，直接调用。

### 5. JobQueue.Size() bug 修复

原版
```go
func (q *LockQueue) Size() int {
    q.lock.RLock()
    defer q.lock.RUnlock()
    return s.Size()  // 错把 s 当 q
}
```
monitorbeat 改为 `return q.jobs.Size()`。

### 6. metrics 删除

原版 `gm.TaskRunTotal.With(taskLabels).Inc()` 等 Prometheus 指标在 monitorbeat 中去掉，P1 阶段以 `internal/stats` 重新建。

## 四、procsource.Source 雏形（P2 预埋）

`internal/host/procsource.go` 提供 `Source` interface 与 3 个 stub：

- `FileSource` — P2 用来读本地 hostid 文件（替代原 `/var/lib/gse/host/hostid` 硬编码）
- `HTTPSource` — P2 支持远端 API 拉身份，适配云环境
- `StaticSource` — P0/P1 测试用，`SetInfo` 可注入任意数据

P0 阶段所有方法返回 `nil` 或 `ErrNotImplemented`，不参与调度链路。P2 实现 procconf/procsync/proccustom 时才接入。
