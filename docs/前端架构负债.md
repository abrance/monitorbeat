# 前端架构负债 (monitorweb)

> monitorweb 前端 (React + Vite + TypeScript) 当前已识别但未解决的架构缺口与需优先处理的问题。
> 排序按影响程度：🔴 生产风险 > 🟡 功能缺失 > 🟢 体验优化。
> 记录时间：2026-07-16。后续新发现的问题追加于此。

---

## 🔴 生产风险

### API 类型声明与运行时不一致

- **现状**：`types.ts` 定义 `ProbeSeries.up`/`duration` 为 `Point[]`（非空数组），但 Go 后端零值序列化为 `null`。前端对 null 无防御 → 页面白屏崩溃
- **状态**：**P0 已修** — `server.go` 初始化空切片 + `Probes.tsx` 可选链防御。但问题未根治
- **风险**：同一个坑会出现在所有 Go `nil slice` 序列化为 `null` 的 API 字段。未来新增 API 端点可能重复此 bug
- **根因**：前后端类型定义由两个团队（或不同时间）维护，无自动同步机制。Go `[]T` 零值为 `nil`，JSON 序列化出 `null`，
  TypeScript 却声明为 `T[]`（不包括 `null`）
- **方案**：
  - 短期：后端所有 `nil slice` 改为 `[]T{}` 初始化（已修 ProbeResult，但 EventsResult、MetricSeries 同样有隐患）
  - 中期：引入 `zod` 运行时校验，API 响应解码时做类型断言 + 默认值
  - 长期：从 Go 结构体自动生成 TypeScript 类型（`go-ts` / `typed-go`）

### 无 Error Boundary

- **现状**：整个 SPA 无 `React.ErrorBoundary`。任何组件渲染时抛错（如本次 pro `null.length`）→ 整个页面空白，用户无法操作
- **影响**：生产环境单点故障。用户看到白屏无任何提示
- **方案**：每个 `<Route>` 外层包裹 `<ErrorBoundary>`，fallback 显示错误信息 + 重试按钮 + 错误详情可复制

### 零前端测试

- **现状**：`web/ui/` 无任何测试文件。无 Vitest / React Testing Library / Playwright
- **影响**：本次 `null.length` 回归未被任何流程捕获。未来每次重构或新功能都可能引入同级 bug 而无人知晓
- **方案**：加 Vitest + RTL，至少覆盖：
  - 每个 page 的 正常/加载/错误/空数据 状态渲染
  - 每个组件 props 边界情况
  - API client 错误处理路径

## 🟡 功能缺失

### 页面无自动刷新

- **现状**：所有页面只在组件挂载时 fetch 一次数据（`useAsync` deps 不变不重新请求）。Probes/Overview/HostDetail 都显示静态快照
- **影响**：用户看不到最新数据，需手动刷新浏览器
- **方案**：`useAsync` 加 `refetchInterval` 参数。Probes 30s 轮询，Overview 60s，HostDetail 60s。考虑后续升级为 SSE / WebSocket

### 无时间范围选择器

- **现状**：所有页面写死 `now - 3600`（1 小时）。Probes/HostDetail 均无快捷范围选择
- **影响**：无法查看历史趋势（24h/7d），无法排查偶发问题
- **方案**：加 `<TimeRangePicker>` 组件，支持 1h/6h/24h/7d 快捷切换 + 自定义起止时间。范围变更时重新请求

### 无加载骨架 / Error 状态

- **现状**：
  - `Probes.tsx` 数据加载中显示空白 section（无骨架/无 loading 指示器）
  - `HostDetail.tsx` metrics 加载时整段显示 "loading…"，无法利用已有统计卡片
  - `Overview.tsx` 无 hosts 数据时直接显示空白网格
- **影响**：用户感知"页面卡死"，无法区分"正在加载"、"无数据"、"出错了"
- **方案**：统一 `Skeleton` 组件。每个 section 独立 loading state。错误时显示 retry 按钮

### useAsync Hook 功能不足

- **现状**：自定义 `useAsync` 是简易版的 `useEffect` + `useState` 封装，无缓存、无去重、无后台刷新
- **影响**：
  - Overview 切换页面回来后重新请求（浪费带宽）
  - 多个组件请求同一端点无去重
  - 无 stale-while-revalidate，每次都是同步阻塞的 loading 切换
- **方案**：
  - 短期：`useAsync` 加缓存层（`Map<key, data>` + TTL）
  - 中期：迁移到 `@tanstack/react-query`（成熟、类型好、缓存+去重+自动刷新）

### 图表性能（O(n*m) 合并）

- **现状**：`Chart.tsx` 将多 series 合并为 chart 数据时，每个时间戳都做 `Array.find()` 线性扫描。`n` 个时间戳 × `m` 个 series = O(n*m)
- **影响**：选择大量 metric（48 个 checkbox）时，页面卡顿
- **方案**：预构建 `Map<timestamp, Record<metric, value>>`，O(n+m) 合并

### 响应式布局缺失

- **现状**：仅网格布局 `auto-fill, minmax(260px, 1fr)` 算是响应式，其余为固定宽度设计，移动端不可用
- **方案**：加 `@media` breakpoint（768px/480px），导航栏折叠 hamburger，stat/card 堆叠布局

## 🟢 体验优化

### TypeScript strict 未全开

- **现状**：`tsconfig.json` 中 `noUnusedLocals: false`，`noUnusedParameters: false`
- **影响**：未使用的 import/变量/参数不会被 TS 检测，积累死代码
- **方案**：改为 `true`，清理现有死代码后永久开启

### Bundle 体积 >500KB

- **现状**：Vite build 警告 `index-DuVLHIWm.js 556.45 kB (gzip 161.74 kB)`，`recharts` 是主要体积来源
- **影响**：首次加载 ~162KB（gzip 后）
- **方案**：`recharts` 改为动态 `import()` 懒加载；或用更轻的 `uPlot` / `lightweight-charts`

### Host 卡片 Stat 块溢出

- **现状**：`.host-stats` 是 3 列网格 `.stat-card` 全局 `min-width: 110px`，卡片压缩时 Disk 块溢出边界
- **修复**：`.host-stats .stat-card { min-width: 0; padding: 10px 8px; }`，跟随网格列宽自适应
- **状态**：**已修**（commit `44d01f3`，重建静态资源后生效）

### 无数据状态不统一

- **现状**：各种空数据展示不统一：`"no probe data"` / `"no data"` / `"—"` / `"loading…"` / 直接空白
- **方案**：统一 `<EmptyState message="..." icon="..." />` 组件

### favicon.ico 404

- **现状**：浏览器请求 `/favicon.ico` 返回 404。无图标
- **方案**：`web/ui/public/favicon.svg` + `manifest.json`

### disk___used_percent 三方下划线

- **现状**：`server.go` 和 `HostDetail.tsx` 中指标名 `disk___used_percent` 含三个连续下划线（Go struct tag `disk___used_percent`）
- **原因**：疑似从 bkmonitorbeat 迁移的命名遗物
- **方案**：统一改为 `disk_used_percent`（Go agent + Go server + Frontend 同步改名，注意 breaking change）

### API 无统一错误协议

- **现状**：各个 handler 错误响应格式不一致（`{error: string}` / `{error: string, status: ...}` / 纯文本）
- **方案**：统一错误响应格式 `{error: string, code?: string}`，前端 `api.get<T>()` 统一解析

### CORS 未配置

- **现状**：monitorweb Go 端无 CORS 中间件，仅同源使用
- **影响**：无法从其他域名 SPA 调用 API
- **方案**：加 CORS middleware，允许配置 `Access-Control-Allow-Origin`

---

## 已标记快速胜 (Quick Wins)

1. **favicon.ico 404** → 丢一个 `favicon.ico` 到 `web/ui/public/`
2. **disk___used_percent** → 改名 `disk_used_percent`（同步 agent → server → frontend）
3. **Error Boundary** → 半小时可加，防止白屏
4. **Probes auto-refresh 30s** → `useAsync` 加 `setInterval`
5. **`noUnusedLocals: true`** → 清死代码开检查
