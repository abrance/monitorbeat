## 🟡 功能缺失

### checker scheduler 未接线

- **现状**：`scheduler/checker/scheduler.go` 已实现并测试通过，但 `main.go` 未接入
- **影响**：一次性执行类 task（如配置同步）无法使用
- **原因**：P2 搁置了 procconf/procsync/proccustom 三个 CMDB-bound task，暂无 checker 场景
- **方案**：等有 checker 类 task 时，在 `main.go` 用 config hint 或 type set 路由到 checker

### cron scheduler 空

- **现状**：`scheduler/cron/` 目录空，cron 表达式调度未实现
- **影响**：无法支持 "每天 3:00 执行" 类定时任务
- **参考**：bkmonitorbeat 用 `github.com/roylee0704/gron`
- **方案**：引入 gron 或 `github.com/robfig/cron`，实现 `scheduler/cron/scheduler.go`

### listen scheduler 空

- **现状**：`scheduler/listen/` 目录空，长连接/监听模式调度未实现
- **影响**：trap (SNMP trap 接收) 和 kubeevent (k8s watch) 无法实现
- **参考**：bkmonitorbeat listen scheduler 基于 fsnotify/端口监听
- **方案**：抽象 `ListenScheduler` 接口：`Listen(ctx, handler)` + `Stop()`，具体 task 实现 handler

### eventbus 多订阅者

- **现状**：`internal/eventbus/` 目录空，当前用裸 `chan define.Event` + engine 同步 fan-out
- **影响**：慢 output 阻塞其他 output；无法独立消费/过滤/缓冲
- **方案**：每个订阅者独立 `chan`，bus 负责广播 + 背压；P1 不急需，可延迟

### internal/stats 空

- **现状**：`internal/stats/` 目录空，selfstats task 用 `runtime.ReadMemStats()` 绕过
- **影响**：无 `/metrics` 端点暴露；无法对接 Prometheus 抓取
- **方案**：引入 `github.com/prometheus/client_golang/promhttp`，在 admin server 挂 `/metrics`

### file output 轮转策略不全

- **现状**：file output 支持 `max_size_mb` 轮转，但无 `max_age` / `max_backups` 清理
- **影响**：旧文件无限堆积，磁盘可能满
- **方案**：加 `MaxAge` / `MaxBackups` 配置项，启动时清理过期文件

### http output 生产特性缺失

- **现状**：http output 支持基础 POST JSON + 本地 fallback，无 gzip/batch/mTLS/OAuth2/熔断
- **影响**：高吞吐场景带宽大；TLS 双向认证不可用
- **方案**：分批（batch 100 条或 1s 窗口）/ gzip 压缩 / mTLS 证书配置 / circuit breaker

### keyword 多文件 glob

- **现状**：keyword `path` 只支持单文件路径，不支持 glob 模式
- **参考**：bkmonitorbeat 支持 `*.log` / `/var/log/**/*.log`
- **方案**：`path` 字段加 `filepath.Glob` 解析，每个匹配文件起独立 harvester

### keyword 聚合模式 (keyword_event)

- **现状**：只实现 `raw_log` 模式（逐行匹配即发），无周期聚合（count + window）
- **参考**：bkmonitorbeat keyword_event 按维度聚合，周期内计数后批量发送
- **方案**：加 `mode: keyword_event` + `report_period`，内存做 dimension 聚合

### keyword multiline / json / 自定义 parser

- **现状**：只做逐行正则匹配，不支持多行日志、JSON 解析、自定义 format
- **参考**：bkmonitorbeat 的 processor 链
- **方案**：`multiline` 按首行正则合并；`format: json` 结构化提取

### script 时间戳解析

- **现状**：script event 使用当前时间，不从 metric 行解析 timestamp
- **参考**：bkmonitorbeat 支持从 prometheus metric line 提取 timestamp
- **方案**：解析 prometheus text 的 timestamp 字段，fallback 到当前时间

### dmesg 非 root 不可用

- **现状**：使用 `os/exec dmesg` 执行，非 root 用户 exit status 1
- **参考**：bkmonitorbeat 用 `/dev/kmsg` tail read (listen 模式)
- **方案**：加 `use_kmsg: true` 模式直读 `/dev/kmsg`（需 root + listen scheduler）；文档注明权限要求
